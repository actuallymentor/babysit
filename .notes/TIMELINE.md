# Timeline

- **2026-04-28**: Project scaffolded. Based on sir-claudius analysis + agent CLI research. Tech: Node.js 24, bun compile, mentie, mri, yaml.
- **2026-04-28**: Full implementation completed across all 10 phases. All 55 tests passing. Discovered mentie `hash()` is async — switched to synchronous `crypto.createHash` in matcher, session, volumes, and credential refresh modules.
- **2026-04-28**: Spec compliance pass — fixed seven concrete bugs and wired up the statusline path that was missing from the v0.1.0 cut. See `.notes/GOTCHAS.md` for the new mri caveat. 73 tests passing.
- **2026-04-28 (later)**: Second compliance pass after audit against SPECIFICATION.md. Highlights: docker image namespace fix (was `babysit/babysit`, now `actuallymentor/babysit`), codex resume switched from non-interactive `exec resume` to interactive `resume`, system prompt support extended to codex/gemini/opencode via env+entrypoint, shell-quoting added to `build_docker_command` (multi-line/special-char system prompts were silently mangled), sandbox+mudbox combination now rejected, dep check + self-update now run on every subcommand, install.sh actually offers to install missing deps, default model/effort filled in for codex per spec ("max effort, latest model"). 88 tests passing.
- **2026-04-28 (later still)**: Third compliance pass — all three bugs broke compiled binary, two of them silently. (1) `--no-update` was silently ignored: mri normalises `--no-X` to `{X:false}` but parse.js read `args['no-update']`. (2) `package.json` and `system_prompt/*.md` were read via `readFileSync(__dirname/...)` — fails at runtime in bun-compiled binaries because `__dirname` resolves to `/$bunfs`. Switched to compile-time imports (JSON attribute for package.json; converted .md fragments to JS-exported string constants in `src/system_prompt/index.js`). (3) Removed dead `get_statusline_path` export which had the same broken pattern. 96 tests passing.
- **2026-04-29**: Fourth compliance pass after re-checking each adapter against current vendor docs. Five concrete bugs, all silent: (1) codex effort was passed as `-c reasoning_effort=high` — the real key is `model_reasoning_effort`, the bare form is silently ignored, so we'd been launching codex at the wrong effort level in every yolo run. (2) codex skip-permissions was `--full-auto`, which leaves codex's workspace-write sandbox active and blocks edits inside our docker sandbox; switched to `--dangerously-bypass-approvals-and-sandbox` for true babysit-yolo equivalence. (3) Default codex model `gpt-5-codex` is not a real model id; updated to `gpt-5.5` (the latest GA frontier model as of April 2026). (4) Default gemini model `gemini-2.5-pro` was outdated; switched to the rolling `gemini-pro-latest` alias. (5) Dockerfile PATH didn't include `~/.local/bin` or `~/.opencode/bin`, which is where the claude/opencode install scripts symlink their binaries — at runtime `claude`/`opencode` would not be on PATH. Also unref()'d the credential refresh interval so the cli can exit cleanly when interrupted before the tmux session closes, and deleted the orphaned `src/modes/{yolo,sandbox,mudbox}.js` helpers (had a never-used context-mutation API). 97 tests passing.
- **2026-04-29 (later)**: Adapter home-dir pinning + a real silent codex bug. Each agent now declares an `home: { env_var, dir }` block (CODEX_HOME, GEMINI_CLI_HOME, CLAUDE_CONFIG_DIR, OPENCODE_CONFIG_DIR) and `build_docker_command` bakes that env var into the docker run — single source of truth for where the agent reads global instructions, and a stray host-side value can no longer redirect the agent to an unmounted path. While auditing this, found that codex's `system_prompt_file` was pointing at `~/.codex/instructions.md` — a legacy filename that current codex no longer reads. The real path is `${CODEX_HOME}/AGENTS.md`, so babysit's system prompt has been silently dropped on codex sessions since v0.3.0. Fixed. 103 tests passing.
- **2026-04-29 (latest)**: Fifth compliance pass — credential coverage audit caught three silent failures, all surfacing as "container launches unauthenticated even though I logged in on the host". (1) darwin opencode creds were dropped because darwin.js only handled keychain_service + fallback_file, never the standalone `file:` field opencode declares. (2) codex OAuth tokens at `~/.codex/auth.json` weren't forwarded — only env-key API auth was supported, so anyone who ran `codex auth login` had no creds in the container. (3) gemini OAuth tokens at `~/.gemini/oauth_creds.json` had the same gap. Each adapter now declares `file:` alongside its env keys, the darwin layer routes standalone files through a new `mount_credential_file` helper, and the Dockerfile pre-creates the four agent dirs so file mounts don't leave them root-owned. Also implemented the pre-flight rotation step (`<agent> --version` between detect and capture) so near-expiry tokens get refreshed by the agent's own logic before we copy. Plus: claude `--sandbox` now skips the writable session dirs (RO-mounting them was crashing claude on first write), and `bun.lock` (Bun 1.2+) is finally recognised as a Node project signal. 110 tests passing.
- **2026-04-29 (later)**: Sixth pass — major UX gap and three smaller bugs. (1) Auto-attach: `babysit claude --yolo` previously left the user at the babysit cli while the monitor ran in the foreground and the agent ran in a detached tmux session — they'd have to open a second terminal and `tmux -L babysit attach` to actually use the agent. The spec ("the user can detach and re-attach to the session as needed") implied auto-attach. Refactored `cmd_start` to spawn `babysit __monitor <id>` as a detached daemon (which re-loads everything from session metadata and runs its own credential sync), then `execSync` `tmux attach` for the user. Detach with Ctrl+B d exits the foreground cli but leaves the daemon + container running. (2) `babysit resume <id>` ignored the original session's pwd — `cmd_resume` delegated to `cmd_start` without `chdir`-ing first, so resuming from a different cwd loaded the wrong babysit.yaml. (3) `config.commands` execution used `execSync`, freezing the supervisor while a slow notify_command was running. Switched to async `spawn`. (4) `cmd_open` interpolated session names directly into a shell string instead of going through the existing JSON.stringify-based `attach_session` helper. Plus: removed the vestigial `creds_sync_pid` field from session metadata (sync runs as `setInterval` in-process, not as a child PID — the field had been `null` since v0.1.0). 113 tests passing.
